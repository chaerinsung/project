Cafes in Los Angeles

#Core Application

Application was built using Laravel

  • Pages
    • If you do not register or login, you only have access to these pages:
      • home
      • register
      • login
    • Once you have registered or logged in, you have accesses to these pages:
      • home
      • cafe
        • This is where you have access to the cafe database. You can add cafes to the database, and/or you can delete or edit cafes that you added.
      • profile
      • logout
  • Server-side validation with Laravel's validation rules
  • Error messages as flashed data for when form submissions fail validation
  • Form submissions that fail validation repopulate the form with the user's input
  • Notifications or Bootstrap-style alerts as flashed data for when inserts, updates, and deletions are successful
  • Authentication - Sign up, Login, and Logout
  • Blade templates that share common layout
  • Use of Eloquent for database access
  • Built a REST API/CRUD

#Feature 1: Comments

Commenting system for each cafe. The comments are unique to that entity.

  • The comments in your commenting system contain the commenter's name, a comment body, and a timestamp.
  • Comments are displayed from the most recent to the oldest.
  • A user should be able to edit and delete their own comments but nobody else's.

#Feature 2: Favorites

Favorites system that allows user to add a cafe to their profile

  • A user should be able to add and remove a particular cafe from their favorites section.
  • When the favorites section is displayed in the profile, the cafe's name and the date/time of when it was favorited is displayed.

#Database

Built a SQLite database and used Laravel migrations
